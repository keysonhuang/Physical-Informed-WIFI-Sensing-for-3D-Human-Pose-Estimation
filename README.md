# Physics-Informed WiFi Sensing for 3D Human Pose Estimation
（This readme file was generated by Claude, just for those who are interested so that they could easily start the code.）

Code for the paper *"Physics-Informed WiFi Sensing for Robust 3D Human Pose Estimation in Mobile and Cross-Environment Settings"*.

---

## Requirements

- Python 3.8+
- PyTorch 2.0+
- `pywt`, `scipy`, `numpy`, `pyyaml`

```bash
pip install torch pywt scipy numpy pyyaml
```

## Dataset

Download the **MM-Fi** and **Person-in-wifi-3D** dataset and place it at:

```
MMFi_Dataset/Dataset/
```

## File Structure

```
├── config.yaml        # Dataset split and training settings

├── mmfi.py            # MM-Fi dataset loader
```

## Configuration

Edit `config.yaml` to select the dataset split:

| `split_to_use`       | Description              |
|----------------------|--------------------------|
| `random_split`       | Random 80/20 split       |
| `cross_subject_split`| S01–S09 train, S10 val   |
| `cross_scene_split`  | E01–E03 train, E04 val   |
| `manual_split`       | A01–A21 train, A22–A27 val |

## Training

```bash
python train.py
```

Checkpoints are saved to `saved_models/`. The best model (lowest val MPJPE) is saved as `best_model.pth`. Only the 10 most recent epoch checkpoints are kept.

---

## Model Overview

The `PhysicsGuidedPoseEstimator` in `model.py` has three components:

- **PGMA** — Transformer encoder that captures multipath-aware CSI features
- **HCAF** — Fuses action semantics with CSI features to resolve pose ambiguity
- **PDDA** — Domain adaptation via gradient reversal to suppress environment-specific features

Input: CSI tensor `(B, T, antennas, subcarriers)` + action label `(B,)`  
Output: 3D pose `(B, 17, 3)` in mm + environment logits `(B, 4)`

---

## Metrics

Evaluated with three standard metrics: **MPJPE**, **PA-MPJPE**, and **PCK@k** (k = 20/30/40/50). All implemented in `evaluation.py`.
